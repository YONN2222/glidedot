# Using Translations

Consume the finalized translations from Glide in your application.

## 1. List Available Languages
Before fetching a translation, you can retrieve a list of all languages configured for your project to know which language codes are available.

### Request
`GET /localization/projects/:projectId/languages`

**Headers:** `Authorization: Bearer <API_KEY>`

### Example Response
```json
[
  {
    "id": 1,
    "code": "en",
    "name": "English",
    "flag": "🇬🇧",
    "isSource": true
  },
  {
    "id": 2,
    "code": "de",
    "name": "German",
    "flag": "🇩🇪",
    "isSource": false
  }
]
```

---

## 2. Fetch Translations (Public Endpoint)
Fetches all approved translations for a specific language as a flat JSON key-value map. 

This endpoint is **100% public** and supports CORS. You can call this directly from the browser inside your frontend application without exposing any API keys.

### Request
`GET /localization/projects/:projectId/translations/:langCode`

- `projectId` (integer): The ID of the project.
- `langCode` (string): The language code (e.g., `en`, `de`).

### Example Response
```json
{
  "header.title": "Welcome to Glide",
  "auth.login.button": "Sign In",
  "errors.not_found": "Page not found"
}
```
